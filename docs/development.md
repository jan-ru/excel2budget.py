# Development Guide

## Prerequisites

- Python 3.12+
- Node.js 18+
- [uv](https://docs.astral.sh/uv/getting-started/installation/) (recommended Python package manager)

## Backend Setup

```bash
cd backend
uv venv
uv sync --all-extras
```

## Frontend Setup

```bash
cd frontend
npm install
```

## Running the Backend

Using the entry point script:

```bash
cd backend
uv run python backend/run.py
```

Or directly with uvicorn (with hot reload for development):

```bash
cd backend
uv run uvicorn backend.app.main:app --reload
```

The OpenAPI spec is served at `http://localhost:8000/openapi.json`.
Prometheus metrics are available at `http://localhost:8000/metrics`.

Override defaults with environment variables:

```bash
HOST=127.0.0.1 PORT=9000 LOG_LEVEL=debug DUCKDB_PATH=/tmp/dev.duckdb uv run uvicorn backend.app.main:app --reload
```

## Type Generation

With the backend running, generate TypeScript types from the OpenAPI spec:

```bash
cd frontend
npm run generate-types
```

This produces `frontend/src/types/api.d.ts`. Do not edit this file manually — it is overwritten on each generation.

## Running Tests

Backend (115 tests — Hypothesis property tests + integration):

```bash
cd backend
uv run pytest tests/ -v
```

Frontend (170 tests — fast-check property tests + integration):

```bash
cd frontend
npx vitest --run
```

## CLI

The backend includes a CLI for dev/ops use. It runs as a one-off process using the same codebase and template registry as the API:

```bash
cd backend

# Convert a file
uv run python -m backend.app.cli input.xlsx twinfield budget --budgetcode BC01 --year 2026

# List packages
uv run python -m backend.app.cli --list-packages

# List templates for a package
uv run python -m backend.app.cli --list-templates twinfield
```

Exit codes: 0 (success), 1 (input/config error), 2 (transform/export error).

## Pre-commit Hooks

The repo uses [pre-commit](https://pre-commit.com/) with local hooks. Install with:

```bash
uvx pre-commit install
uvx pre-commit install --hook-type pre-push
```

On every commit:
- Ruff lint + format check on backend Python files
- TypeScript type check (`tsc --noEmit`) on frontend files
- `api.d.ts` staleness check when backend Pydantic models change

On every push:
- Full backend test suite (`pytest`)
- Full frontend test suite (`vitest`)

Run all hooks manually:

```bash
uvx pre-commit run --all-files
```
